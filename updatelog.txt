v0.4 — 2026-05-09  (All todo features — date picker, compare, flow diagram, etc.)
--------------------------------------------------------------------------------
ADDED
  - Animated SVG power flow diagram (Live section) — directional dash animation
    showing energy moving between Solar, Battery, Home, and Grid in real time
  - Push alerts — browser notification when grid import exceeds configurable watt
    threshold; bell toggle (🔔) in Live header; 5-minute cooldown between alerts
  - Date picker toolbar in Reports — Daily uses date input, Weekly uses any date
    in the target week, Monthly uses month picker, Yearly uses year number input
  - Comparison mode — "Compare" button fetches previous period (prev day/week/
    month/year) and overlays as dashed lines / lighter bars on all charts
  - CSV export — /api/export/csv?period=X&date=Y returns a spreadsheet with date,
    solar kWh, grid kWh, savings, CO₂ offset per row; "⬇ CSV" button in toolbar
  - PDF export — "🖨 PDF" button calls window.print(); print CSS hides nav and
    shows all sub-panels for a clean printable layout
  - Monthly Bill Estimator (Savings section) — projects monthly grid bill, bill
    without solar, and monthly saving based on this month's avg daily usage
  - Cumulative savings area chart (Savings section) — running total of money saved
    month-by-month this year using Chart.js gradient area fill
  - Mobile PWA — static/manifest.json + static/sw.js service worker caches app
    shell; icons in static/icon-192.svg and static/icon-512.svg
  - Responsive improvements — viewport-fit=cover, tighter padding on mobile, grid
    collapses correctly at 500px, nav wraps on small screens
  - Alert threshold setting — configurable watt threshold for grid alert notifications
  - Timezone setting — stored in settings.json, IANA name (e.g. Asia/Karachi)
  - /api/export/csv Flask endpoint — streams CSV response with Content-Disposition
  - /api/stations Flask endpoint — lists available power stations for account
  - /api/report-data now accepts ?date=YYYY-MM-DD to shift the reference date for
    all period calculations (daily/weekly/monthly/yearly relative to that date)

CHANGED
  - DEFAULT_SETTINGS expanded: alert_threshold (1000W), timezone (Asia/Karachi)
  - settings_api now persists alert_threshold and timezone fields
  - Live section header shows alert bell button beside station name
  - Settings page shows Station ID in system info card

v0.1 — 2026-05-09  (Initial release)
--------------------------------------------------------------------------------
ADDED
  - Flask web app with SEMS Portal (GoodWe) API integration
  - Login page with email/password authentication
  - Session-based credential storage (browser session only)
  - Report page with 4 tabs: Daily / Weekly / Monthly / Yearly

  - Chart.js 4.4 for all visualisations
